## Определение

`Maps` (или словари) — это наборы пар ключ-значение.  
Ключи уникальны, и каждый из них соответствует одному значению,  
в то время как значения могут дублироваться.

В общем случае `Map` содержит _только_ элементы _одного_ типа,  
но типы ключей и значений могут различаться: например, `Duck` для ключей и `Int` для значений.  
В некоторых случаях возможно хранение элементов разных типов,  
но между ними должна существовать _связь_, подробнее смотрите в [документации](https://kotlinlang.org/docs/generics.html).

![Определение Map](../../utils/src/main/resources/images/duck/shop/theory/map_definition.png)

<div class="hint" title="Нажмите, чтобы узнать о различных реализациях Map">

Аналогично случаям с `List` и `Set`, вы можете использовать различные реализации `Map`, чтобы изменить это свойство:  
например, вы можете использовать реализацию `LinkedHashMap`, которая предоставляется по умолчанию.  
`LinkedHashMap` — это двусвязный HashMap, который сохраняет порядок вставки при  
итерации. Он использует асимптотическую `O(1)` для добавления и чтения по ключу  
и полагается на функцию `hashCode()` элементов, как и реализация `LinkedHashSet`.
</div>

## Инициализация

Чтобы создать новую карту, можно использовать специальные _строители_:

```kotlin
val emptyMap1 = emptyMap<Int, String>() // Создает внутренний объект EmptyMap
val emptyMap2 = mapOf<Int, String>() // Вызывает emptyMap()
```

<div class="hint" title="Нажмите, чтобы узнать разницу между emptyMap и стандартной картой">

В Kotlin реализация `emptyMap` под капотом является объектом.  
Основная причина существования такой функции — экономия на выделениях (создание и удаление новых переменных).  
Поскольку `emptyMap` возвращает один и тот же _синглтон_ каждый раз, когда вызывается,  
можно использовать его без выделения новых ресурсов. Это позволяет вашей программе на Kotlin работать более эффективно.
</div>

```kotlin
val mapOfNumbers = mapOf(1 to "one", 2 to "two") // Тип может быть выведен, создается набор с элементами 1 to "one", 2 to "two"
```
