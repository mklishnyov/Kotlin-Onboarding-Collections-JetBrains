Мы уже научились готовить томатный суп, теперь пришло время приготовить овощной салат и увидеть разницу между списком (list) и последовательностью (sequence)!
В этом задании мы сделаем функциональной кнопку `Salad list`.

**Обратите внимание**, что в этом задании _набор доступных кнопок в приложении отличается_.
В наличии будут только те кнопки, которые необходимы для выполнения этого задания.
Это ожидаемое поведение.

В этом задании вам нужно реализовать три функции в файле `Salad.kt`:

- `getFreshVegetables` - получает все овощи из холодильника (см. `FridgeImpl.getAllVegetables`) и возвращает только свежие
- `cut` - это функция-расширение для списка овощей (из функции `getFreshVegetables`), которая готовит овощи для салата:

    - выложите каждый овощ на кухню (см. `KitchenImpl.put`)
    - нарежьте каждый овощ (см. `KitchenImpl.cut`)
    - затем возьмите `NUM_VEGETABLES_FOR_SALAD` из списка
    - и наконец возьмите эти овощи с кухни (см. `KitchenImpl.take`)

- `mixVegetablesForSalad` - принимает список нарезанных овощей и готовит их:

    - группирует овощи по их типам
    - затем добавляет каждый из них в салатницу (см. `SaladBowlImpl.add`)
    - в завершение перемешивает овощи (см. `SaladBowlImpl.mix`)

Вы могли заметить, что иногда у вас остаются нарезанные овощи на столе.
Это происходит потому, что вы использовали _список_ для их хранения, и каждое действие применялось к каждому элементу этого списка.
Но в итоге мы взяли только часть овощей, достаточную для салата.

Если в этом коде изменить всего одну строку и использовать _последовательность_ вместо _списка_, порядок выполнения действий изменится автоматически.
